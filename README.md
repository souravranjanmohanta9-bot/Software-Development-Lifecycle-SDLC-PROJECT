# Software-Development-Lifecycle-SDLC-PROJECT
1.Manual Testing: It is the process of manually checking software for defects without using any automated tools.
SDLC(Software Development Life Cycle): A step-by-step process used by teams to design, develop, and test high-quality software.
STLC(Software Testing Life Cycle): A sequence of specific activities conducted during the testing process to ensure software quality goals are met.
Edge Case: A problem or situation that occurs only at an extreme (maximum or minimum) operating parameter.
Bug/Defect: Any mismatch between the "Expected Result" and the "Actual Result" found during testing.
2. Testing Scenarios ()
"Happy Path" vs "Edge Case" 
Sourav: Term English Definition Example (Login Page)
Happy Path Testing the system with valid inputs and expected behavior. Entering a correct email and password to log in successfully.
Edge Case Testing boundary conditions or unusual scenarios. Trying to log in with an extremely long password (e.g., 500 characters).
Negative Testing Testing the system with invalid inputs to see if it handles errors correctly. Entering a wrong password to see if an error message appears.
3. The Bug Report Format
Agar aapko koi bug milta hai, toh use English mein aise report karein:
Defect ID: Unique number for the bug.
Summary: A short description of the issue.
Steps to Reproduce: Step-by-step instructions to find the bug.
Severity: Impact of the bug (Low, Medium, High, Critical).
Priority: How fast the bug needs to be fixed.
Expected Result: What the software should do.
Actual Result: What the software actually did.
